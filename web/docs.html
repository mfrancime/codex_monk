▣ CODEX_MONK what is this ← WAR ROOM

An evolvable, declarative agentic OS

behavior is data — swarms of tiny agents, gated by DNA, watching the machine

codex_monk is a miniature operating system for swarms of monitoring agents. Each agent watches some surface of a machine — kernel memory pressure, per-pod cgroups, disk and network, a Kubernetes control plane — and decides a severity. What makes it unusual is how those decisions are expressed: not as code, but as a short string of genome opcodes that an interpreter runs against live readings. New behavior is a new genome string and a line of YAML, never a new program.

▎ The design law

New behavior comes from a new genome string + new YAML, never a new Python class. There is exactly one agent class and one genome interpreter; every capability is recombined from opcodes and config. Adding code is a deliberate, named tradeoff — not the default.

01The fabric — a shared-memory world

Every swarm lives in a single fixed-size fabric: a 512 KB shared-memory file (/dev/shm/codex.<name>.fabric) holding agent control blocks, a 1024-slot key/value state table, per-agent inboxes, and an event log. Agents are real OS processes that meet only here — they post their verdicts to state, message each other through inboxes, and leave a trail in the log. The format is byte-compatible with the swarm it forked from, which is why one swarm can read another's mind (see the governor below).

02The genome — alien DNA, not code

A genome is a compact UTF-8 string of stack-based (RPN) opcodes, interpreted against a Frame (one tick of a probe's readings) plus a per-domain opcode table, yielding a (severity, code) verdict. It's deliberately crash-proof: unknown or mutated opcodes simply no-op, so a garbled genome can only score poorly — it can never take an agent down. That property is what lets the system evolve its own rules by random mutation.

03The agent — one class, three roles

There is a single DeclarativeAgent. Its role is chosen by config, never by subclass:

  • probe — has a genome: samples its domain, runs the interpreter, and emits on every verdict change.
  • sink — has consume_types: persists the alerts it receives into a timeline.
  • mutator — has a target: evolves a genome by trying random mutations, keeping only improvements, and writing the winner back — even across swarms. The swarm rewrites its own DNA while running.

04The probes — sensor surfaces

A probe is a small module that turns a slice of the machine into a Frame and claims a disjoint opcode letter. Adding a domain = adding one probe module; the behavior on top of it is still pure genome + YAML.

05Multiswarm — composing a cluster

Many swarms run side by side, one OS process each, every one with its own fabric. They bridge through a gateway agent speaking VJR — an HMAC-signed, length-prefixed JSON protocol — so a node's alert can travel to a central aggregator. The k8s monitor composes three:

k8s_node──▶ k8s_aggregator◀── k8s_cluster

Nodes watch the host; the cluster swarm watches the control plane; the aggregator is the central timeline they all report to.

06The governor — judgement across swarms

The aggregator runs a governor: a quorum probe that opens every sibling fabric, reads each one's verdict, and turns per-swarm signals into a single cluster-wide judgement. Its whole policy is one genome:

Γc0>Γk∧→Cd ; Γk0≡→Cg ; Γp0>→Wp

a node CRITICAL while the control plane is healthy → CLUSTER_DEGRADED; the control plane itself unreadable → GATE_DOWN; a node merely pressured → POD_PRESSURE.

Notably this needed no new opcode — cross-fabric aggregation lives in the probe, and the stateless genome just gates on the resulting counts.

07The war room — seeing it live

The page you came from is a full-screen 3D view of all this in motion: each swarm is a glowing hub, agents orbit it, gateway links pulse with traffic, and the whole scene reddens as the global DEFCON drops. Hubs are colored by severity — OK, WARN, CRITICAL — reading straight from the same fabrics the agents write to. It is a window onto live shared memory, not a mock-up.
